# VibeCoding Edu Tool - 전체 프로젝트 구상 계획서

## 1. 프로젝트 개요 (Overview)
이 프로젝트는 AI 시대의 새로운 개발 패러다임인 '바이브 코딩(Vibe Coding)'과 전통적인 '소프트웨어 엔지니어링 역량'을 동시에 가르치는 하이브리드 교육 플랫폼입니다. 비개발자, 어린이부터 전문가까지 누구나 자연어 프롬프트를 통해 코드를 생성하는 법을 배우는 동시에, 생성된 코드가 내부적으로 어떻게 동작하는지(코드 문해력), 예외 상황(Edge Case)은 어떻게 처리해야 하는지(시스템 아키텍처)를 리버스 엔지니어링 방식으로 학습합니다.

## 2. 타겟 페르소나 엔진 (Target Persona Engine)
사용자의 수준에 따라 AI의 설명 깊이와 은유(Metaphor)가 동적으로 변화합니다.
*   **어린이/학생 (Kids/Students):** 직관적인 비유(레고 블록, 요리 등)와 논리적 흐름(Flow) 중심. "왜 이렇게 지시해야 컴퓨터가 알아듣는지"에 집중.
*   **입문자 (Beginners - 비개발자):** 전문 용어를 배제하고 프론트/백/DB의 큰 그림(Architecture) 이해에 주력. 에러 발생 시 AI에게 묻는 방법 학습.
*   **중급자/경력자 (Intermediates - 주니어 개발자):** 코드의 동작 원리를 명확히 설명하고, 성능 최적화 및 보일러플레이트 제거 방법에 집중.
*   **전문가 (Experts - 시니어 개발자):** 아키텍처 설계, 보안 취약점, 최신 트렌드/대안 기술과의 비교 및 고급 디자인 패턴을 강제하는 정교한 프롬프팅.

> **Phase 1 페르소나 지원:** 모든 트랙(VSCode Extension, Claude Code Plugin, CLI 분석기)은 모든 페르소나에게 개방됩니다. 다만 레벨에 따라 **권장 진입 경로**가 다릅니다:
>
> | 페르소나 | 권장 트랙 | 사유 |
> |---------|----------|------|
> | **학생 (청소년기+)** | VSCode Extension | GUI 기반, 터미널 불필요, Copilot Free 티어 활용 가능 |
> | **입문자** | VSCode Extension | Marketplace에서 원클릭 설치, 시각적 호버 설명 |
> | **중급자** | VSCode Extension + Claude Code Plugin | 기존 워크플로우 내 실시간 교육 |
> | **전문가** | 전체 3트랙 (+ CLI 분석기) | 세션 히스토리 사후분석, 가장 깊은 인사이트 |
>
> **어린이(Kids)**는 애니메이션과 시각적 다이어그램으로 접근성을 극대화할 수 있는 Phase 2(Web GUI)에서 지원.
>
> 입문자/학생이 터미널을 한 번도 열지 않고 사용할 수 있도록 **"Getting Started" 가이드**를 스크린샷과 함께 제공: VSCode 설치 → Copilot Free 활성화 → Extension 설치 (3단계).

## 3. 핵심 차별화 기능 (Core Differentiating Features)
기존의 단순 프롬프트 생성 툴이나 문법 위주의 코딩 교육 플랫폼과 차별화되는 이 프로젝트만의 독창적인 기능들입니다. 구현 우선순위와 기술적 실현 가능성에 따라 **MVP(Phase 1)**과 **향후(Phase 2+)**로 구분합니다.

### Phase 1 — MVP 기능

#### 3.1. 엑스레이 뷰어 (Code X-Ray Viewer) — **코드 문해력 특화** (MVP 핵심)
*   **기능:** AI가 생성한 코드의 각 줄이 '왜' 필요한지 전통적인 프로그래밍 관점에서 설명. IDE에서는 호버(Hover) 또는 인라인 주석으로, CLI 분석기에서는 세션 히스토리 기반 구조화된 리포트로 제공.
*   **교육 효과:** 바이브 코딩이 만들어낸 '블랙박스'를 해체하여, 변수, 반복문, 비동기 처리 등 전통적인 코드의 문법과 구조를 스스로 읽을 수 있는 눈(Code Literacy)을 길러줍니다.

#### 3.2. 프롬프트 평가기 (Prompt Evaluator) (MVP 핵심)
*   **기능:** 사용자의 프롬프트를 입력받아 원본 코드와의 의도/구조적 유사도를 채점하고 실행 가능한 피드백을 출력.
*   **교육 효과:** 모호한 지시가 아닌 정확하고 잘 구조화된 프롬프트 작성법을 가르침.

#### 3.3. "Fix the AI's Mess" — 버그 해결 도장 (Debugging Dojo) (MVP 핵심)
*   **기능:** 시스템이 의도적으로 논리적 오류(Logic Bug)나 비효율적인 스파게티 코드를 포함한 결과물을 제시.
*   **교육 효과:** 사용자는 코드를 읽고 오류를 찾아낸 뒤, "처음부터 다시 짜"가 아니라 "이 for문을 map으로 바꾸고 예외 처리를 추가해"와 같이 AI를 디버깅하는 구체적인 프롬프트를 작성하는 실전 훈련을 합니다.

#### 3.4. 재실행 비교 (Re-run Diff) — 비결정성 교육 (MVP 부가 기능)
*   **기능:** 동일한 프롬프트를 같은 모델에서 재실행하여 두 결과물의 차이를 하이라이트.
*   **교육 효과:** AI가 생성한 코드는 비결정적(non-deterministic)이라는 사실 — 같은 프롬프트도 매번 다른 결과를 낼 수 있음 — 을 가르쳐, 맹목적 신뢰가 아닌 코드 리뷰와 이해의 필요성을 강화.
*   **비용:** 비교 1회당 모델 호출 1회 추가 (사용자 선택).

### Phase 2+ — 향후 기능

#### 3.5. "Vibe to Logic" 번역기 (Prompt Anatomy & Translation)
*   **기능:** 사용자가 자연어(Vibe)로 입력한 프롬프트가 실제 코드의 어떤 '기술적 컴포넌트'로 번역되는지 시각적으로 매핑.
*   **교육 효과:** 모호한 지시가 시스템 구조로 치환되는 과정을 눈으로 확인하며, 전통적인 컴퓨팅 사고력(Computational Thinking)을 기릅니다.
*   **Phase 2+ 사유:** 다이어그램, 애니메이션 등 풍부한 시각적 UI가 필요하여 Web GUI에 적합.

#### 3.6. 보안 및 엣지 케이스 시뮬레이터 (Security & Edge-Case Simulator)
*   **기능:** 사용자가 프롬프트로 완성한 정상 작동 코드에 시스템이 고의로 '악의적 입력(예: SQL Injection)'이나 '극단적 상황(예: 동시 접속 1만 명)'을 주입하여 코드를 터뜨림.
*   **교육 효과:** "작동하는 것"에 만족하지 않고, 시스템이 망가지는 조건을 상상하여 이를 방어하는 코드(Defensive Programming)를 프롬프트에 추가하도록 유도.
*   **Phase 2+ 사유:** 시각적 시뮬레이션 결과와 인터랙티브 UI가 있어야 교육적 효과 극대화.

#### 3.7. 단계별 빌드업 튜토리얼 (Step-by-Step Architecture Build)
*   **기능:** 큐레이션된 오픈소스 프로젝트를 분석하여 10개 내외의 작은 기능적 단계(예: 1. DB 스키마, 2. API, 3. UI 렌더링)로 쪼개줌.
*   **교육 효과:** 거대한 소프트웨어를 한 번의 마법 같은 프롬프트로 만드는 것이 아니라, 모듈 단위로 분해(Decomposition)하여 조립하는 아키텍처 설계 능력을 기릅니다.
*   **범위:** 큐레이션된 예제 프로젝트 3~5개로 한정. 임의 GitHub URL 분석은 현 LLM의 미지 저장소에 대한 안정성 한계로 후순위.

#### 3.8. 멀티 모델 프롬프트 비교 (Multi-Model Prompt Comparison)
*   **기능:** 사용자가 작성한 동일한 프롬프트가 Gemini, Claude, GPT-4 등 서로 다른 모델에서 어떻게 다르게 해석되고 코딩되는지 비교.
*   **교육 효과:** 특정 모델에 종속되지 않고, LLM의 특성을 이해하여 범용적으로 통하는 강건한 프롬프팅 스킬을 배움.
*   **Phase 2+ 사유:** 토큰 비용이 모델 수만큼 배수로 증가. Web GUI에서 명시적 비용 경고와 함께 opt-in 유료 기능으로 제공.

## 4. 아키텍처 및 배포 전략 (Architecture & Deployment Strategy)

### 4.1. 핵심 설계 원칙: 독립형 앱이 아닌 플러그인
본 프로젝트는 AI 모델을 직접 호출하는 **독립형 애플리케이션이 아닙니다.** 사용자가 이미 인증/과금을 마친 **기존 AI 코딩 도구의 플러그인/확장**으로 동작합니다. 이를 통해 API 키 관리, 인증, 과금 인프라 구축이 불필요합니다.

```
사용자의 기존 환경 (모델 비용 = 사용자 부담)
┌──────────────────────────────────────┐
│  VSCode / Cursor / Claude Code /     │
│  Codex CLI                           │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  VibeCoding Edu Plugin         │  │
│  │  ─────────────────────         │  │
│  │  · X-Ray Viewer                │  │
│  │  · Prompt Evaluator            │  │
│  │  · Debugging Dojo              │  │
│  │  · Re-run Diff                 │  │
│  └────────────────────────────────┘  │
│                                      │
│  호스트 도구의 AI 모델 사용 ──────────│
└──────────────────────────────────────┘
```

### 4.2. Phase 1 — MVP (3개 병렬 트랙)

**Track A: VSCode Extension** — IDE 내 실시간 교육
*   타겟: 가장 큰 사용자 풀 (수천만). Cursor(VSCode fork)와도 호환.
*   AI 모델 접근: 사용자의 기존 Copilot 구독 또는 API 키.
*   플러그인 시스템: 성숙한 Extension API, 문서화 우수.

**Track B: Claude Code Plugin** — 터미널 내 실시간 교육
*   타겟: 빠르게 성장 중인 AI 네이티브 개발자 커뮤니티.
*   AI 모델 접근: 사용자의 기존 Anthropic 구독 (이미 인증 완료).
*   플러그인 시스템: Hook/Skill 시스템 (신생).

**Track C: CLI 사후분석 도구** — 완료된 프로젝트의 오프라인 분석
*   기존 로컬 프로젝트의 AI 세션 히스토리 파일을 파싱하여 교육용 리포트를 생성.
*   **Level 1 (AI 모델 불필요, 무료, 오프라인):**
    *   프롬프트 → 코드 변경 매핑 (어떤 프롬프트가 어떤 파일을 바꿨는지)
    *   코드 구조 트리 (함수/클래스/모듈 관계)
    *   변경 빈도 히트맵 (AI가 가장 많이 수정한 영역)
    *   프롬프트 패턴 통계 (명확한 지시 vs 모호한 지시)
*   **Level 2 (사용자 자체 API 키, 선택적):**
    *   X-Ray 해설 (각 코드 블록의 "왜")
    *   보안/엣지케이스 진단
    *   프롬프트 개선 제안 ("이렇게 썼으면 더 나은 코드가 나왔을 것")
    *   실제 세션 데이터 기반 Debugging Dojo 문제 자동 생성

**세션 파일 지원 (Phase 1):**

| 도구 | 세션 파일 위치 | 형식 | 프로젝트 매핑 방식 |
|------|-------------|------|-----------------|
| **Claude Code** | `~/.claude/projects/<인코딩된-경로>/세션ID.jsonl` | JSONL | 디렉토리명에 프로젝트 경로가 인코딩 — 정확한 매핑 가능 |
| **Codex CLI** | `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl` | JSONL | JSONL 내부 `working_directory` 필드를 파싱하여 프로젝트 매칭 |

### 4.3. 공유 모노레포 구조

3개 트랙 모두 TypeScript 모노레포를 통해 코어 로직을 공유합니다:

```
vibecoding-edu/
├── packages/
│   ├── core/                    ← 공유 핵심 로직
│   │   ├── xray-parser.ts             # 코드 해부 엔진
│   │   ├── prompt-evaluator.ts        # 프롬프트 채점
│   │   ├── debugging-dojo.ts          # 버그 주입/검증
│   │   ├── diff-analyzer.ts           # 재실행 비교
│   │   └── session-parsers/
│   │       ├── claude-code.ts         # Claude Code JSONL 파서
│   │       └── codex-cli.ts           # Codex CLI JSONL 파서
│   │
│   ├── vscode-extension/        ← Track A: VSCode/Cursor 어댑터
│   │   └── extension.ts
│   │
│   ├── claude-code-plugin/      ← Track B: Claude Code 어댑터
│   │   ├── hooks/
│   │   └── skills/
│   │
│   └── cli-analyzer/            ← Track C: CLI 사후분석 도구
│       └── cli.ts
```

### 4.4. Phase 2 — Web GUI
*   Next.js 기반 웹 애플리케이션으로 어린이/학생 및 입문자 접근성 극대화.
*   시각적 기능: Vibe to Logic 번역기, 보안 시뮬레이터 (아키텍처 다이어그램 및 애니메이션).
*   멀티 모델 비교: opt-in 유료 기능으로 제공.
*   LMS 연동 (Canvas, Google Classroom) — B2B 교육용.

### 4.5. Phase 3 — 확장
*   Gemini CLI Extension (Google 생태계).
*   추가 세션 파서 (Aider, Cline/Roo Code 등).
*   수요에 따른 추가 IDE 연동.

## 5. 경쟁사 분석 (Competitive Landscape Analysis)

### 5.1. 시장 개요

현재 시장은 두 진영으로 명확히 분리되어 있으며, 그 교차점을 점유한 플레이어가 존재하지 않습니다:

```
┌─────────────────────────────────────────────────┐
│           AI 코드 생성 (Make)                     │
│     Replit  ·  bolt.new  ·  v0  ·  Cursor        │
│                      ↑                            │
│                ┌─────┴─────┐                      │
│                │ VibeCoding │  ← 유일한 교차점     │
│                │  Edu Tool  │                      │
│                └─────┬─────┘                      │
│                      ↓                            │
│           코드 이해 / 교육 (Learn)                 │
│     Codecademy  ·  Scratch  ·  Code.org           │
└─────────────────────────────────────────────────┘
```

**한 줄 포지셔닝:** *"AI가 만든 코드를 이해하는 유일한 교육 플랫폼"*

### 5.2. 경쟁사 비교 매트릭스

| 서비스 | 핵심 성격 | 타겟 | 가격 | AI 교육 요소 | 코드 문해력 |
|--------|----------|------|------|-------------|------------|
| **Replit Agent** | AI 코드 생성 + 실행환경 | 초보~중급 개발자 | $0–100/월 | 코드 생성 과정 설명 | ✗ |
| **Cursor** | AI 네이티브 IDE | 중급~고급 개발자 | $0–200/월 | ✗ (생산성 도구) | ✗ |
| **bolt.new** | 프롬프트 → 풀스택 앱 | 비개발자~초급 | $0–200/월 | ✗ (결과물 중심) | ✗ |
| **v0.dev** | 프롬프트 → React UI | 프론트엔드 개발자 | $0–30/월 | ✗ (결과물 중심) | ✗ |
| **Copilot Workspace** | GitHub 이슈 → PR | 중급~고급 개발자 | $10–39/월 | ✗ (워크플로우 도구) | ✗ |

### 5.3. 경쟁사별 차별점

| 비교 대상 | 본 프로젝트의 차별점 |
|----------|-------------------|
| **Replit / bolt / v0** | 이들은 코드를 "만들어주고 끝". 본 프로젝트는 만들어진 코드를 **해부(X-Ray)**하고 **스트레스 테스트(Simulator)**하여 이해시킴. |
| **Cursor / Copilot** | 전문 개발자 생산성 도구로 교육 목적이 아님. 본 프로젝트는 **코드가 왜 이렇게 구조화되었는지** 가르침. |
| **Codecademy** | 전통 문법 교육에 AI를 보조로 붙인 것. 본 프로젝트는 **AI 생성 코드를 역분석**하는 새로운 학습 방향을 제시. |

### 5.4. 핵심 기회: 블루오션

기존 제품 중 **"만들기 → 이해하기(Make → Understand)" 루프**를 점유한 제품은 없습니다:
*   AI 코드 생성 도구는 만들어주지만, 만들어진 것이 무엇인지 가르치지 않습니다.
*   전통 교육 플랫폼은 문법을 가르치지만, 빠르게 표준이 되고 있는 AI 기반 워크플로우를 무시합니다.
*   본 프로젝트는 **교육 목적의 AI 결과물 역분석**이라는 유일한 교차점에 위치합니다.

### 5.6. 전략적 권고

1.  **"만들기 → 이해하기(Make → Understand)" 루프**를 핵심 마케팅 메시지로 — 경쟁자가 점유하지 않은 포지션.
2.  생성 도구와 **경쟁이 아닌 파트너십** — 예: "bolt로 만든 앱, VibeCoding으로 이해하기." Replit/bolt/v0와의 연동을 성장 채널로 활용.
3.  **B2B 교육 시장을 장기 타겟**으로 — 바이브 코딩이 주류가 되면서 대학과 부트캠프에 "AI 시대 코드 문해력" 커리큘럼이 필요해질 것.

## 7. MVP 마일스톤 (MVP Milestones)

### Track A: VSCode Extension
1.  **Extension 스캐폴드:** TypeScript 기반 VSCode Extension 프로젝트 셋업, activation events 및 command palette 설정.
2.  **X-Ray Viewer:** AI 생성 코드를 줄 단위로 해설하는 호버(Hover)/인라인 주석 기능 구현.
3.  **Prompt Evaluator:** 에디터 사이드바 내 프롬프트 채점 패널 연동.

### Track B: Claude Code Plugin
1.  **Plugin 스캐폴드:** Claude Code 플러그인 프로젝트를 hooks 및 skills 구조로 셋업.
2.  **X-Ray Skill:** 가장 최근 AI 생성 코드 블록을 분석하는 `/xray` 스킬 구현.
3.  **Debugging Dojo Skill:** 코드에 버그를 주입하고 프롬프트 기반 수정을 연습하는 `/dojo` 스킬 구현.

### Track C: CLI 사후분석 도구
1.  **세션 파서:** Claude Code 및 Codex CLI 세션 파일의 JSONL 파서 구현 (프로젝트 경로 매칭 포함).
2.  **Level 1 정적 분석:** AI 모델 없이 프롬프트→코드 매핑 리포트, 코드 구조 트리, 변경 빈도 히트맵 생성.
3.  **Level 2 AI 심층 분석:** 선택적 사용자 API 키를 통한 X-Ray 해설, 프롬프트 개선 제안, 실제 세션 데이터 기반 Debugging Dojo 문제 자동 생성.

### 공유 코어 (Shared Core)
1.  **모노레포 셋업:** 공유 `core` 패키지가 포함된 TypeScript 모노레포 초기화.
2.  **코어 엔진:** xray-parser, prompt-evaluator, debugging-dojo, diff-analyzer를 재사용 가능한 모듈로 구현.
3.  **Re-run Diff:** 전체 트랙에 걸쳐 X-Ray Viewer 내에 "다시 실행" 비교 기능 추가.
